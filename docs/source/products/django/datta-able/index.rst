`Datta Able </product/datta-able/django/>`__
=============================================

.. title:: Django Datta Able - Open-Source Django Template 
.. meta::
    :description: Open-Source Django Template crafted on top of Datta Able Design
    :keywords: django, starter, django template, datta able, bootstrap 4, django template

**Open-Source Django Template** built with a minimum set of features on top of **Datta Able**, a modern dashboard design from CodedThemes. 
This template can be used to start a new project quickly by adding new features on top of the existing ones or simply for learning purposes.

Features 
--------

- Simple, Easy-to-Extend codebase
- `Datta Able </docs/templates/bootstrap/datta-able.html>`__ Full Integration 
- `Bootstrap </docs/templates/bootstrap.html>`__ CSS Styling 
- `Dynamic Tables <https://django-datta.onrender.com/dynamic-dt/>`__ (demo) - read `docs </docs/developer-tools/dynamic-datatables.html>`__
- `Dynamic API <https://django-datta.onrender.com/api/>`__ (demo) - read `docs </docs/developer-tools/dynamic-api.html>`__
- `Django CLI Package </docs/developer-tools/django-cli/index.html>`__
    - `Commit/rollback Git Changes </docs/developer-tools/django-cli/git-interface.html>`__
    - `Backup & restore DB`
    - `Interact with Django Core </docs/developer-tools/django-cli/query-django.html>`__
    - `Manage Environment`
    - `Manage Dependencies`
- Session-based Authentication, Password recovery
- DB Persistence: SQLite (default), can be used with MySql, PgSql
- Docker, CI/CD for Render
- `Vite </docs/technologies/vite/index.html>`__ for assets management

Download Source Code 
--------------------

The product can be downloaded from the `official product page </product/datta-able/django/>`__ or directly from GitHub (public repository)

.. code-block:: shell

    git clone https://github.com/app-generator/django-datta-able.git
    cd django-datta-able

Once the source code is unzipped, the next step is to start it and use provided features.

Codebase  
--------

The project is coded using a simple and intuitive structure presented below:

.. code-block:: bash   

    < PROJECT ROOT >
      |
      |-- config/                            
      |    |-- settings.py          # Project Configuration  
      |    |-- urls.py              # Project Routing
      |    |-- views.py             # Controllers & Views
      |
      |-- cli/                      # CLI Package
      |    |-- common.py              
      |    |-- h_util.py              
      |    |-- h_shell.py              
      |    |-- h_files.py              
      |    |-- h_django.py              
      |
      |-- apps/
      |    |-- pages/               # Serve UI pages 
      |         |-- views.py         
      |         |-- models.py       # Define project models         
      |         |-- urls.py         
      |    |-- dyn_api/             # Dynamic API Module 
      |    |-- dyn_dt/              # Dynamic Tables Module 
      |
      |-- static/
      |    |-- css                  # CSS Files 
      |    |-- js                   # JS Libraries
      |
      |-- templates/
      |    |-- layouts              # Master pages 
      |    |-- includes             # Components
      |    |-- pages                # Pages 
      |
      |-- requirements.txt          # Project Dependencies
      |
      |-- env.sample                # ENV Configuration (default values)
      |-- manage.py                 # Start the app - Django default start script
      |
      |-- ************************************************************************
